# Screenshots

This document is reserved for screenshots of the live portfolio website.

Screenshots can make the repository look more complete because they show the visual result of the project without exposing private WordPress files.

## Recommended screenshots

1. Home page desktop view
2. Home page mobile view
3. About or profile section
4. Projects section
5. Reviews section
6. Contact section

## How screenshots should be added

Screenshots should be clear, cropped properly and saved with simple file names.

Good examples are:

1. home desktop
2. home mobile
3. projects section
4. reviews section
5. contact section

## Suggested folder

Screenshots can be placed inside an assets folder.

Example structure:

assets/screenshots

## Notes

Only public parts of the website should be shown.

Private dashboards, WordPress admin pages, analytics, email addresses, login pages and hosting panels should not be included.
