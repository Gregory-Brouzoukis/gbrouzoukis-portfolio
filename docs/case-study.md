# Portfolio Website Case Study

## Overview

The portfolio website was created to build a clear online presence for Gregory Brouzoukis.

The purpose of the project is to present technical interests, personal work, programming progress and future projects in one place.

## Problem

A personal profile needs more than a simple social media page.

It needs a clear structure, a professional design, useful information and a way to connect different parts of someone’s technical work.

The website was created to solve this by becoming the central place for projects, contact information, GitHub work and future development.

## Solution

The website was built with WordPress and Elementor, then improved with custom styling, SEO setup and cleaner structure.

The design uses a dark visual style with orange accent color, focused sections and clear calls to action.

## Main features

1. Personal introduction
2. Web development section
3. Programming and applications section
4. AI automation section
5. GitHub and technical work section
6. Contact information
7. Reviews area
8. Legal and trust pages

## Tools used

1. WordPress
2. Elementor
3. Custom CSS
4. Hostinger
5. Papaki
6. Google Search Console
7. GitHub

## What was learned

This project helped with understanding how a website is structured, how SEO works, how Google indexing needs time and how important it is to keep public pages clean and organized.

It also showed the importance of mobile design, legal pages, search appearance and project documentation.

## Next steps

The next steps are to add stronger project pages, upload screenshots, improve mobile details and connect the website more clearly with GitHub repositories.
